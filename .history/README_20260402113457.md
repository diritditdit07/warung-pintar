# Warung Pintar Kasir

Aplikasi kasir PWA sederhana untuk warung makanan kecil. Fokusnya adalah layar yang besar, tombol besar, dan pemakaian offline tanpa login.

## Fitur

- Dashboard dengan untung hari ini
- Halaman jual dengan tombol produk besar dan keranjang sederhana
- Halaman pengeluaran untuk mencatat biaya harian
- Laporan harian: total penjualan, total pengeluaran, dan untung
- Data tersimpan di `localStorage`
- Bisa di-install sebagai PWA dan tetap berjalan offline setelah aset aplikasi tersimpan

## Menjalankan

```bash
npm install
npm run dev
```

Untuk akses dari HP di jaringan Wi-Fi yang sama:

```bash
npm run dev:host
```

Atau untuk preview hasil build dari HP:

```bash
npm run build
npm run preview:host
```

Build produksi:

```bash
npm run build
```

## Akses dari HP dan PWA

- Jika hanya ingin membuka aplikasi dari HP saat laptop dan HP ada di Wi-Fi yang sama, gunakan `npm run dev:host` atau `npm run preview:host`, lalu buka alamat IP laptop dari browser HP.
- Jika ingin install sebagai PWA di HP, yang paling aman adalah deploy sementara ke HTTPS, misalnya Vercel.
- Link lokal berbentuk `http://192.168.x.x:5173` biasanya bisa dibuka di HP, tetapi install PWA dan service worker sering tidak aktif penuh karena browser mobile mensyaratkan secure context (`https`) untuk PWA.
- Jadi:
  - testing tampilan di HP: bisa pakai link lokal
  - install aplikasi PWA di HP: lebih baik deploy ke Vercel dulu

## Deploy ke Vercel

Karena ini app Vite statis, deploy ke Vercel cukup sederhana:

```bash
npm install -g vercel
vercel
```

Atau hubungkan repository ke Vercel lalu gunakan setting default berikut:

- Framework Preset: `Vite`
- Build Command: `npm run build`
- Output Directory: `dist`

Setelah deploy, buka URL HTTPS dari HP lalu pilih `Add to Home Screen` atau `Install App` dari browser.

## Struktur Folder

```text
src/
  components/   Komponen UI per layar
  data/         Data awal produk
  utils/        Format uang, tanggal, dan penyimpanan lokal
  App.jsx       Alur aplikasi utama
  main.jsx      Entry point React + registrasi PWA
supabase/
  sql/          Schema database dan helper function
  functions/    Edge Function untuk login kode warung + PIN
```

## Integrasi Supabase

Fondasi backend custom auth sudah disiapkan di folder `supabase/`. Lihat `supabase/README.md` untuk langkah SQL dan deploy Edge Function.

Untuk frontend, buat file `.env` berdasarkan `.env.example` lalu isi:

```bash
VITE_SUPABASE_URL=...
VITE_SUPABASE_ANON_KEY=...
```

Status saat ini:

- Login `kode warung + PIN` sudah terhubung ke Supabase Edge Function
- Session login sudah disimpan di browser
- Data lokal aplikasi sekarang dipisah per warung di browser
- Sinkronisasi `products`, `sales`, `expenses`, dan `settings` ke Supabase belum disambungkan penuh dan menjadi langkah berikutnya

## Pengembangan Lanjutan

- Tambahkan menu edit produk di `src/data/defaultProducts.js` atau pindahkan ke pengaturan lokal baru
- Jika data makin besar, pindahkan penyimpanan dari `localStorage` ke IndexedDB tanpa mengubah UI utama
- Jika ingin printer struk atau sinkronisasi cloud, buat adapter baru di `src/utils/`
