# Supabase Setup

Project ini disiapkan untuk login `kode warung + PIN` menggunakan Supabase Edge Functions.

## Yang Perlu Dilakukan

1. Buka Supabase SQL Editor.
2. Jalankan file `supabase/sql/schema.sql`.
3. Deploy edge function `warung-login`.
4. Pastikan env di project Vercel berisi:
   - `VITE_SUPABASE_URL`
   - `VITE_SUPABASE_ANON_KEY`

## Deploy Edge Function

```bash
supabase login
supabase link --project-ref xukjjzyrqzcyqyhhqntt
supabase functions deploy warung-login
```

Edge function ini membutuhkan secret default berikut di project Supabase:

- `SUPABASE_URL`
- `SUPABASE_SERVICE_ROLE_KEY`

## Demo Login Awal

Setelah SQL dijalankan, data demo awal akan tersedia:

- Kode warung: `WARUNG-DEMO`
- PIN: `1234`

Segera ganti atau hapus data demo ini setelah testing.

## Jika Sebelumnya SQL Gagal di Bagian `crypt()`

Schema terbaru sudah tidak memakai `crypt()` lagi. Sekarang hash PIN memakai kombinasi `pin_salt + md5`, jadi lebih kompatibel untuk SQL Editor Supabase.

Kalau kamu sempat menjalankan schema versi lama sebagian, jalankan lagi file `supabase/sql/schema.sql` yang terbaru ini. Untuk user demo, script akan reset ulang PIN demo ke `1234`.

Kalau kamu sudah sempat membuat user custom sebelum schema terbaru dipakai, paling aman hapus user itu lalu buat ulang dengan skema hash terbaru.

## Catatan Penting

- Dengan model `kode warung + PIN`, frontend tidak boleh query tabel bisnis langsung memakai anon key.
- Semua akses sensitif sebaiknya lewat Edge Function agar session token custom bisa diverifikasi aman.
- File di folder `supabase/` ini adalah fondasi backend. Frontend app masih perlu dihubungkan ke flow login dan CRUD berbasis function/token.
